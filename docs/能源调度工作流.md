\# 能源调度工作流设计报告



> \*\*文档版本\*\*：v1.0

> \*\*编制日期\*\*：2026-06-17

> \*\*关联项目\*\*：能耗智能管理优化平台

> \*\*设计目标\*\*：构建基于扣子平台的能源调度优化工作流，实现基于分时电价的生产排程智能决策





\## 一、 业务背景与需求



\### 1.1 业务场景



在工厂能耗管理场景中，生产调度员每天面临的核心挑战是：\*\*如何在满足生产交付的前提下，尽可能降低用电成本\*\*。



工业用电普遍实行\*\*分时电价\*\*政策（峰、平、谷），各时段电价差异显著（典型峰谷价差可达 4 倍）。调度员需要决定：

\- 哪些设备在什么时间运行？

\- 如何将高能耗任务集中安排在谷电时段？

\- 峰电时段安排哪些低能耗操作（维护、待机）？



\*\*能源调度智能体\*\*解决的核心问题：



| 业务痛点 | 智能体解决方案 |

|---------|-------------|

| 峰谷电价差异大，人工排程难以全局最优 | 算法自动按电价从低到高分配生产任务 |

| 多设备产能不同，任务分配复杂 | 按设备产能比例智能分配运行时长 |

| 缺乏量化评估，无法衡量调度效果 | 自动计算优化前后成本对比，量化节费金额 |

| 调度方案难以向管理层说明逻辑 | 大模型生成自然语言推理说明 |

\## 二、 工作流整体设计



\### 2.1 设计原则



| 原则 | 说明 |

|------|------|

| \*\*成本最优\*\* | 优先利用谷电时段，最大限度降低用电成本 |

| \*\*产能约束\*\* | 确保在截止时间前完成生产目标 |

| \*\*设备适配\*\* | 按设备产能比例分配任务，避免单设备过载 |

| \*\*结果可量化\*\* | 输出预估节省金额，让优化价值看得见 |

\### 3.2 参数说明



| 参数字段 | 类型 | 必填 | 来源 | 说明 |

|---------|------|------|------|------|

| `production\_goal` | integer | 是 | 前端输入 | 必须完成的生产目标（件数） |

| `deadline` | string | 否 | 前端输入 | 截止时间（ISO 8601），不传默认当日 23:59:59 |

| `available\_devices` | array | 是 | 系统 `devices` 表 | 用户指定的可用设备列表 |

| `available\_devices\[].id` | integer | 是 | `devices.id` | 设备ID |

| `available\_devices\[].name` | string | 是 | `devices.name` | 设备名称 |

| `available\_devices\[].rated\_power` | number | 是 | `devices.rated\_power` | 额定功率(kW) |

| `available\_devices\[].run\_power` | number | 是 | `rated\_power × 0.85` | 正常运行功率(kW) |

| `available\_devices\[].idle\_power` | number | 是 | `rated\_power × 0.15` | 待机功率(kW) |

| `tariff\_policy` | array | 是 | 系统 `tariff\_policy` 表 | 当前激活的分时电价策略 |

| `tariff\_policy\[].period` | string | 是 | 时段名称 | 高峰/平段/低谷 |

| `tariff\_policy\[].start` | string | 是 | 开始时间(HH:MM) | 时段起始 |

| `tariff\_policy\[].end` | string | 是 | 结束时间(HH:MM) | 时段结束 |

| `tariff\_policy\[].price` | number | 是 | 电价(元/kWh) | 该时段电价 |



\### 3.3 数据来源映射



```

系统数据库                         工作流入参

─────────────────────────────────────────────────

前端输入                         → production\_goal

前端输入                         → deadline



devices.id                      → available\_devices\[].id

devices.name                    → available\_devices\[].name

devices.rated\_power             → available\_devices\[].rated\_power

rated\_power × 0.85              → available\_devices\[].run\_power

rated\_power × 0.15              → available\_devices\[].idle\_power



tariff\_policy (is\_active=True)

&#x20; period\_name                   → tariff\_policy\[].period

&#x20; start\_time                    → tariff\_policy\[].start

&#x20; end\_time                      → tariff\_policy\[].end

&#x20; price\_per\_kwh                 → tariff\_policy\[].price

```



\### 3.4 run\_power 与 idle\_power 计算说明



| 字段 | 计算公式 | 业务含义 |

|------|---------|---------|

| `run\_power` | `rated\_power × 0.85` | 设备正常运行时的典型功率（85%负载率） |

| `idle\_power` | `rated\_power × 0.15` | 设备待机/空转时的典型功率（15%负载率） |



> 注：系数 0.85 和 0.15 为通用估算值，实际项目中可配置化或由用户自定义。

\### 5.2 代码节点 — 调度优化引擎



\#### 5.2.1 节点定位



代码节点是工作流的\*\*核心计算引擎\*\*，负责在满足产能约束的前提下，按电价从低到高分配生产任务，生成最优调度计划并计算节费金额。



\#### 5.2.2 输入变量



| 变量名 | 类型 | 引用方式 |

|--------|------|---------|

| `production\_goal` | Number | `{{start.production\_goal}}` |

| `available\_devices` | Array\[Object] | `{{start.available\_devices}}` |

| `tariff\_policy` | Array\[Object] | `{{start.tariff\_policy}}` |



\#### 5.2.3 核心算法逻辑



```

1\. 按电价从低到高排序 tariff\_policy

2\. 计算总产能（所有设备 run\_power 之和 / 10）

3\. 计算所需总运行时长 = production\_goal / 总产能

4\. 按电价从低到高依次分配任务：

&#x20;  a. 计算本时段可排产时长（不超过剩余需求，且单时段上限8小时）

&#x20;  b. 按各设备产能比例分配本时段运行时长

&#x20;  c. 生成调度记录（设备、时段、动作）

&#x20;  d. 扣减剩余需求

5\. 计算预计节省成本：

&#x20;  a. 优化成本 = Σ(设备功率 × 运行时长 × 时段电价)

&#x20;  b. 基准成本 = 所有任务在最高电价时段运行

&#x20;  c. 节省 = 基准成本 - 优化成本

```



\#### 5.2.4 完整代码



```python

def main(production\_goal, available\_devices, tariff\_policy):

&#x20;   """

&#x20;   能源调度优化代码节点

&#x20;   

&#x20;   输入参数：

&#x20;     - production\_goal: int 生产目标（件数）

&#x20;     - available\_devices: \[ { id, name, rated\_power, run\_power, idle\_power }, ... ]

&#x20;     - tariff\_policy: \[ { period, start, end, price }, ... ]

&#x20;   

&#x20;   输出：

&#x20;     - schedule: Array<Object> 调度计划

&#x20;     - estimated\_cost\_saved: float 预计节省成本（元）

&#x20;   """

&#x20;   

&#x20;   # ========== 1. 辅助函数 ==========

&#x20;   def calc\_end\_time(start, duration\_hours):

&#x20;       """计算结束时间（仅处理小时级偏移，分钟简化为00）"""

&#x20;       if not start or duration\_hours <= 0:

&#x20;           return start

&#x20;       start\_hour = int(start.split(":")\[0])

&#x20;       end\_hour = start\_hour + int(duration\_hours)

&#x20;       if end\_hour >= 24:

&#x20;           end\_hour = end\_hour % 24

&#x20;       return f"{end\_hour:02d}:00"

&#x20;   

&#x20;   # ========== 2. 参数校验 ==========

&#x20;   if production\_goal <= 0:

&#x20;       return {"schedule": \[], "estimated\_cost\_saved": 0}

&#x20;   

&#x20;   if not available\_devices or len(available\_devices) == 0:

&#x20;       return {"schedule": \[], "estimated\_cost\_saved": 0}

&#x20;   

&#x20;   if not tariff\_policy or len(tariff\_policy) == 0:

&#x20;       return {"schedule": \[], "estimated\_cost\_saved": 0}

&#x20;   

&#x20;   # ========== 3. 按电价从低到高排序 ==========

&#x20;   sorted\_tariffs = sorted(tariff\_policy, key=lambda x: x\["price"])

&#x20;   

&#x20;   # ========== 4. 计算总产能和所需总运行时长 ==========

&#x20;   # 假设：每台设备每小时产能 = run\_power / 10（简化模型）

&#x20;   total\_capacity\_per\_hour = sum(\[d\["run\_power"] / 10 for d in available\_devices])

&#x20;   required\_hours = production\_goal / total\_capacity\_per\_hour

&#x20;   

&#x20;   # ========== 5. 按电价从低到高分配任务 ==========

&#x20;   schedule = \[]

&#x20;   remaining\_hours = required\_hours

&#x20;   

&#x20;   # 计算各设备产能占比

&#x20;   device\_capacity\_ratios = \[

&#x20;       (d\["run\_power"] / 10) / total\_capacity\_per\_hour 

&#x20;       for d in available\_devices

&#x20;   ]

&#x20;   

&#x20;   for tariff in sorted\_tariffs:

&#x20;       if remaining\_hours <= 0:

&#x20;           break

&#x20;       

&#x20;       # 本时段最多排产小时数（限制单时段不超过8小时）

&#x20;       max\_hours\_in\_period = 8

&#x20;       hours\_this\_period = min(remaining\_hours, max\_hours\_in\_period)

&#x20;       

&#x20;       if hours\_this\_period < 0.5:

&#x20;           continue

&#x20;       

&#x20;       # 本时段内，各设备按产能比例分配运行时长

&#x20;       for idx, device in enumerate(available\_devices):

&#x20;           device\_hours = hours\_this\_period \* device\_capacity\_ratios\[idx]

&#x20;           if device\_hours < 0.5:

&#x20;               continue

&#x20;           

&#x20;           end\_time = calc\_end\_time(tariff\["start"], device\_hours)

&#x20;           

&#x20;           schedule.append({

&#x20;               "device\_id": device\["id"],

&#x20;               "device\_name": device\["name"],

&#x20;               "start": tariff\["start"],

&#x20;               "end": end\_time,

&#x20;               "action": "run",

&#x20;               "price\_per\_kwh": tariff\["price"]

&#x20;           })

&#x20;       

&#x20;       remaining\_hours -= hours\_this\_period

&#x20;   

&#x20;   # ========== 6. 如果任务未分配完，追加平段补足 ==========

&#x20;   if remaining\_hours > 0:

&#x20;       # 取电价最高的时段作为兜底（理论上不会触发，但做安全处理）

&#x20;       fallback\_tariff = sorted\_tariffs\[-1] if sorted\_tariffs else tariff\_policy\[0]

&#x20;       for idx, device in enumerate(available\_devices):

&#x20;           device\_hours = remaining\_hours \* device\_capacity\_ratios\[idx]

&#x20;           if device\_hours < 0.5:

&#x20;               continue

&#x20;           schedule.append({

&#x20;               "device\_id": device\["id"],

&#x20;               "device\_name": device\["name"],

&#x20;               "start": fallback\_tariff\["start"],

&#x20;               "end": calc\_end\_time(fallback\_tariff\["start"], device\_hours),

&#x20;               "action": "run",

&#x20;               "price\_per\_kwh": fallback\_tariff\["price"]

&#x20;           })

&#x20;   

&#x20;   # ========== 7. 计算预计节省成本 ==========

&#x20;   # 优化方案：各时段实际成本

&#x20;   total\_cost\_optimized = 0

&#x20;   for item in schedule:

&#x20;       start\_hour = int(item\["start"].split(":")\[0])

&#x20;       end\_hour = int(item\["end"].split(":")\[0])

&#x20;       if end\_hour < start\_hour:

&#x20;           end\_hour += 24

&#x20;       duration = end\_hour - start\_hour

&#x20;       if duration <= 0:

&#x20;           continue

&#x20;       

&#x20;       # 获取该设备的 run\_power

&#x20;       device = next((d for d in available\_devices if d\["id"] == item\["device\_id"]), None)

&#x20;       if not device:

&#x20;           continue

&#x20;       power = device\["run\_power"]

&#x20;       total\_cost\_optimized += power \* duration \* item\["price\_per\_kwh"]

&#x20;   

&#x20;   # 基准方案：所有任务在最高电价时段运行

&#x20;   max\_price = max(\[t\["price"] for t in tariff\_policy])

&#x20;   total\_hours = sum(\[d\["run\_power"] / 10 for d in available\_devices])

&#x20;   total\_cost\_baseline = total\_hours \* max\_price

&#x20;   

&#x20;   estimated\_cost\_saved = round(total\_cost\_baseline - total\_cost\_optimized, 2)

&#x20;   if estimated\_cost\_saved < 0:

&#x20;       estimated\_cost\_saved = 0

&#x20;   

&#x20;   return {

&#x20;       "schedule": schedule,

&#x20;       "estimated\_cost\_saved": estimated\_cost\_saved

&#x20;   }

```



\#### 5.2.5 输出变量



| 变量名 | 类型 | 说明 |

|--------|------|------|

| `schedule` | Array\[Object] | 调度计划列表，每项含 device\_id, device\_name, start, end, action, price\_per\_kwh |

| `estimated\_cost\_saved` | Number | 预计节省成本（元） |



\#### 5.2.6 算法约束说明



| 约束项 | 设定值 | 业务依据 |

|--------|--------|---------|

| 单时段最大排产小时 | 8 小时 | 避免单次生产时间过长，保障设备维护间隔 |

| 最小排产小时阈值 | 0.5 小时 | 低于30分钟的生产任务不具排产价值 |

| 低负载阈值判断 | 待机功率 = 额定 × 15% | 工业通用估算值 |





\### 5.3 大模型节点 — 推理说明生成



\#### 5.3.1 节点定位



大模型节点负责将代码节点计算出的调度方案转化为\*\*通俗易懂的业务语言\*\*，让管理人员理解调度策略的核心理念。



\#### 5.3.2 输入变量



| 变量名 | 引用方式 |

| `estimated\_cost\_saved` | `{{code.estimated\_cost\_saved}}` |



\#### 5.3.3 Prompt 模板



```

你是一位工业能源调度专家。请根据以下调度方案生成简洁的推理说明。



\## 调度方案

{{code.schedule}}



\## 预计节省成本

{{code.estimated\_cost\_saved}} 元



\## 输出要求
```

\## 九、 调度示例



\### 示例场景



\- \*\*生产目标\*\*：1000 件

\- \*\*可用设备\*\*：空压机A-1（75kW）、注塑机B-1（55kW）

\- 预计节省成本：150.00 元
