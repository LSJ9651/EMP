# 报告订阅通知配置

## 概述

报告订阅支持三种通知方式，用户可在创建/编辑订阅时选择并配置：

| 通知方式 | 值 | 配置组件 | 说明 |
|---------|-----|---------|------|
| 系统通知 | `system` | 无需配置 | 报告生成后写入 `notifications` 表，通过消息中心查看 |
| 邮件 | `email` | [EmailNotifyConfig.vue](../frontend/src/components/EmailNotifyConfig.vue) | 通过 SMTP 发送邮件通知 |
| 钉钉 | `dingtalk` | [DingtalkNotifyConfig.vue](../frontend/src/components/DingtalkNotifyConfig.vue) | 通过钉钉机器人 Webhook 推送 |

## 数据存储

通知配置以 JSON 字符串存储在 `report_subscriptions.notify_config` 字段中（TEXT 类型）。

### Email 配置 JSON Schema

```json
{
  "smtp_server": "smtp.example.com",
  "smtp_port": 587,
  "sender_email": "noreply@example.com",
  "auth_username": "",
  "auth_password": "your-smtp-auth-code",
  "use_tls": true,
  "email_template": ""
}
```

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `smtp_server` | string | 是 | `""` | SMTP 服务器地址（如 smtp.example.com） |
| `smtp_port` | number | 是 | `587` | SMTP 端口号（25/465/587） |
| `sender_email` | string | 是 | `""` | 发送邮箱地址 |
| `auth_username` | string | 否 | `""` | 认证用户名（留空则与 sender_email 相同） |
| `auth_password` | string | 是 | `""` | SMTP 认证密码或授权码 |
| `use_tls` | boolean | 否 | `true` | 是否启用 TLS 加密传输 |
| `email_template` | string | 否 | `""` | 邮件正文模板，支持 `{{report_name}}`、`{{report_summary}}`、`{{report_time}}` |

### DingTalk 配置 JSON Schema

```json
{
  "webhook_url": "https://oapi.dingtalk.com/robot/send?access_token=xxx",
  "access_token": "",
  "msg_template": "",
  "notify_frequency": "immediately"
}
```

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `webhook_url` | string | 是 | `""` | 钉钉机器人 Webhook 完整 URL |
| `access_token` | string | 否 | `""` | 敏感访问令牌（已含在 URL 中则留空） |
| `msg_template` | string | 否 | `""` | 消息模板，支持 `{{report_name}}`、`{{report_summary}}`、`{{report_time}}`、`{{device_count}}` |
| `notify_frequency` | string | 否 | `"immediately"` | 通知频率：`immediately`(即时) / `batch`(批量) / `daily_summary`(每日摘要) |

## 前端动态显示逻辑

在订阅新建/编辑对话框中，当用户选择通知方式时会动态显示对应配置组件：

```
通知方式: [下拉选择]
  ├─ 系统通知 → 无配置区域
  ├─ 邮件     → <EmailNotifyConfig /> 表单
  └─ 钉钉     → <DingtalkNotifyConfig /> 表单
```

切换通知方式时自动清空旧配置数据。

## API 接口

### 创建订阅

```http
POST /api/agent/subscriptions
Content-Type: application/json

{
  "name": "日报订阅",
  "report_type": "daily",
  "cron_time": "08:00",
  "notify_method": "email",
  "notify_config": {
    "smtp_server": "smtp.example.com",
    "smtp_port": 587,
    "sender_email": "noreply@example.com",
    "auth_password": "secret"
  }
}
```

### 更新订阅

```http
PUT /api/agent/subscriptions/1
Content-Type: application/json

{
  "notify_method": "dingtalk",
  "notify_config": {
    "webhook_url": "https://oapi.dingtalk.com/robot/send?access_token=xxx"
  }
}
```

### 查询订阅列表（响应）

```json
{
  "code": 200,
  "data": {
    "items": [
      {
        "id": 1,
        "name": "日报订阅",
        "notify_method": "email",
        "notify_config": {
          "smtp_server": "smtp.example.com",
          "smtp_port": 587,
          "sender_email": "noreply@example.com"
        }
      }
    ]
  }
}
```

## 前端状态指示

订阅列表表格的"通知方式"列会根据 `notify_config` 是否存在显示状态标签：

| 条件 | 显示 |
|------|------|
| 通知方式 = `system` | 只显示"系统通知" |
| 通知方式 ≠ `system` 且有 `notify_config` | 显示"邮件/钉钉" + 绿色"已配置"标签 |
| 通知方式 ≠ `system` 且无 `notify_config` | 显示"邮件/钉钉" + 黄色"未配置"标签 |

## 数据库迁移

```sql
ALTER TABLE report_subscriptions ADD COLUMN notify_config TEXT;
```

该字段可为 NULL，兼容已有订阅记录。
