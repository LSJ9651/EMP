\## 5. 核心模块设计



\### 5.1 后端核心模块



\#### main.py — 应用入口



```python

\# 职责：初始化 FastAPI 应用、注册路由、启动后台模拟线程

\# 关键内容：

\#   1. 注册所有路由

\#   2. 启动 data\_simulator 后台线程（每5秒生成能耗数据）

\#   3. 启动定时分析任务（每小时触发一次能耗分析智能体）

\#   4. 优雅关闭处理

```



\#### services/data\_simulator.py — 数据模拟引擎



```python

\# 职责：模拟设备能耗数据，持续写入 telemetry 表

\# 核心功能：

\#   - 定义设备状态机（启动预热、稳定运行、待机、关机）

\#   - 基于正弦波 + 随机噪声生成功率值

\#   - 支持“故障注入”：如模拟设备老化导致效率下降

\#   - 每5秒为每个在线设备生成一条能耗记录

```



\#### services/agent\_adapter.py — 智能体适配器



```python

\# 职责：封装与外部智能体的交互（当前为 mock 实现，保留真实接口）

\# 核心方法：

\#   async def analyze\_energy(data: dict) -> dict

\#       输入：设备ID、时间段、能耗原始数据

\#       输出：异常检测结果、节能建议、根因分析

\#   async def optimize\_schedule(context: dict) -> dict

\#       输入：电价表、生产任务、设备功率特性

\#       输出：优化后的设备启停甘特图数据

\# 当前阶段：返回 mock 结构化结果，便于前端联调

```



\#### routers/agent.py — 智能体接口



```python

\# 职责：暴露两个前端调用接口，内部调用 agent\_adapter

\# 端点：

\#   POST /api/agent/analyze  → 能耗分析

\#   POST /api/agent/optimize → 调度优化

\# 请求体结构见 API 设计章节

```

\### 6.2 核心数据表



| 表名 | 说明 | 主键 | 关键字段 |

|------|------|------|---------|

| `devices` | 耗能设备主数据 | `id` | name, type, rated\_power, status |

| `telemetry` | 实时能耗时序数据 | `id` | device\_id, timestamp, power, energy\_kwh |

| `tariff\_policy` | 分时电价策略 | `id` | period\_name, start\_time, end\_time, price\_per\_kwh |

| `alert\_thresholds` | 告警阈值配置 | `id` | device\_id, param\_type, upper\_limit, lower\_limit |

| `alert\_records` | 告警历史记录 | `id` | device\_id, alert\_time, value, message |

| `agent\_reports` | 智能体分析/调度结果 | `id` | report\_type, output\_json, created\_at |

| `schedule\_tasks` | 调度优化建议下发 | `id` | device\_id, start\_time, end\_time, action\_type |



\### 6.3 初始化数据



```python

\# scripts/init\_db.py 会创建上述所有表，并写入：

\# 1. 设备数据：3-5台典型设备（空压机、注塑机、冷水机组、照明）

\# 2. 电价策略：典型峰平谷时段（高峰 1.2元，平段0.8元，低谷0.3元）

\# 3. 告警阈值：每台设备的功率上限（额定功率\*1.1）、下限（额定功率\*0.3空转）

\# 4. 初始模拟 telemetry 数据：过去24小时逐分钟数据（由 init\_mock\_data.py 生成）

```

\## 7. API 接口设计



\### 7.1 REST API 规范



```

请求格式：Content-Type: application/json

认证方式：暂不实现（内网系统，后续可加 Token）

响应格式：{ "code": 200, "data": {...}, "message": "success" }
```

\### 7.2 接口目录（约 25 个端点）



| 模块 | 路径前缀 | 端点示例 | 说明 |

| \*\*看板\*\* | `/api/dashboard` | GET /overview, GET /energyflow, GET /trend | 聚合数据 |

| \*\*报告\*\* | `/api/reports` | GET /export/excel | 报表导出 |

\## 8. 前端架构设计



\### 8.1 页面路由结构



| 路径 | 组件 | 说明 |

|------|------|------|

| `/` | Dashboard.vue | 能耗看板（首页） |

| `/devices` | Devices.vue | 设备管理 |

| `/tariff` | Tariff.vue | 电价策略配置 |

| `/alerts` | Alerts.vue | 告警阈值与记录 |

| `/reports` | AgentReports.vue | 智能体报告列表与详情 |

| `/schedule` | ScheduleOptimize.vue | 调度优化与甘特图 |



\### 8.2 轮询策略



```javascript

// Dashboard.vue 中

const POLLING\_INTERVAL = 5000   // 5秒刷新实时功率

const OVERVIEW\_INTERVAL = 10000 // 10秒刷新概览数据



// 使用 onUnmounted 清除定时器
```

\### 8.3 能流图实现



基于 ECharts `graph` 类型，动态从 `/api/dashboard/energyflow` 获取数据，支持鼠标悬浮显示功率数值，节点颜色区分层级。



\### 8.4 调度甘特图



使用 ECharts 自定义系列或第三方库 `vue-ganttastic`，展示 `/api/agent/optimize` 返回的调度计划，支持手动调整（预留后续功能）。

\## 9. 部署方案



\### 9.1 开发环境



```powershell

\# 后端

cd backend

python -m venv .venv

.venv\\Scripts\\activate

pip install -r requirements.txt

python scripts/init\_db.py

python scripts/init\_mock\_data.py   # 生成历史模拟数据

uvicorn main:app --reload --port 8080



\# 前端

cd frontend

npm install

npm run dev   # 访问 http://localhost:5173

```



\### 9.2 生产环境（单服务器）



```

Nginx :80 → 前端静态文件 (frontend/dist)

Nginx /api → 后端 127.0.0.1:8080

systemd 托管后端 uvicorn 进程

SQLite 数据库定时备份

```



\### 9.3 环境变量



```bash

\# .env 示例

DATABASE\_URL=sqlite:///./energy\_optimizer.db

DEBUG=false

AGENT\_ANALYZE\_URL=https://your-agent.com/analyze   # 预留，当前 mock

AGENT\_OPTIMIZE\_URL=https://your-agent.com/optimize

```
