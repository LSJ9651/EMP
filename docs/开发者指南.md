## 2. 项目结构详解

### 后端分层

```
backend/
├── main.py              # FastAPI 应用工厂，CORS、路由注册、生命周期
├── database.py           # SQLAlchemy session/engine 管理
├── models/               # 数据模型（按领域拆分）
│   ├── __init__.py       # 统一导出（from models import *）
│   ├── base.py           # declarative_base()
│   ├── user.py           # User, UserPermission
│   ├── device.py         # Device, DeviceSynonym
│   ├── alert.py          # AlertThreshold, AlertRecord
│   ├── energy.py         # Telemetry, TariffPolicy
│   ├── ai.py             # 6 个 AI 相关模型
│   └── subscription.py   # ReportSubscription, Notification
├── routers/              # API 路由层（13 个模块）
│   └── *.py              # 每个模块一个文件
├── services/             # 业务逻辑层（13 个服务）
│   └── *.py              # 封装复杂逻辑
├── middleware/
│   └── permission.py     # 角色权限中间件
├── utils/
│   ├── crypto.py         # bcrypt 密码工具
│   └── logger.py         # 统一日志工厂
├── scripts/
│   ├── init_db.py        # 数据库初始化
│   ├── init_mock_data.py # 模拟数据生成
│   └── verify_apis.py    # API 验证脚本
└── tests/                # 后端测试
    ├── test_api.py
    ├── test_notify_config.py
    └── fixtures/         # 测试数据
```

### 路由层约定

- 每个路由文件定义一个 `APIRouter`，通过 `prefix` 和 `tags` 分组
- 只读端点（GET）不强制认证
- 变更端点（POST/PUT/DELETE）通过 `dependencies=[Depends(require_permission("action"))]` 注入权限
- 响应统一使用 `{"code": 200, "data": ...}` 格式

### 服务层约定

- 服务函数接受 `db: Session` 作为第一个参数
- 不直接访问 HTTP 请求/响应对象
- 复杂逻辑拆分为多个服务函数
- 外部 API 调用（Coze）通过专用客户端封装

## 4. 添加新的 API 端点

### Step 1：创建路由文件

在 `backend/routers/` 中创建新文件：

```python
# backend/routers/example.py
from fastapi import APIRouter, Depends
from sqlalchemy.orm import Session
from database import get_db

router = APIRouter(prefix="/api/example", tags=["示例模块"])

@router.get("/")
def list_examples(db: Session = Depends(get_db)):
    return {"code": 200, "data": []}

@router.post("/", dependencies=[Depends(require_permission("manage_example"))])
def create_example(data: ExampleCreate, db: Session = Depends(get_db)):
    return {"code": 201, "message": "创建成功"}
```

### Step 2：定义 Pydantic 模型

在路由文件中定义请求/响应模型：

```python
from pydantic import BaseModel

class ExampleCreate(BaseModel):
    name: str
    description: str = ""
```

### Step 3：注册路由

在 `backend/main.py` 中添加：

```python
from routers.example import router as example_router
app.include_router(example_router)
```

### Step 4：添加前端 API 方法

在 `frontend/src/api/api.js` 中添加：

```javascript
export function getExamples() {
  return api.get('/api/example/')
}

export function createExample(data) {
  return api.post('/api/example/', data)
}
```

## 7. 本地规则引擎扩展

本地模式的核心在 `backend/services/agent_adapter.py` 中的 `_local_*` 方法。

扩展步骤：

1. 在 `agent_adapter.py` 中添加新的 `_local_xxx()` 方法
2. 在 `_dispatch()` 方法中添加新的分发逻辑
3. 基于阈值检查（如空压机功率 > 90% 限额 → 建议降载）
4. 基于时段电价（峰时段 → 建议错峰）

示例：

```python
def _local_new_feature(self, device_ids: list, db: Session):
    devices = db.query(Device).filter(Device.id.in_(device_ids)).all()
    results = []
    for d in devices:
        # 基于规则的分析逻辑
        if d.efficiency < 0.8:
            results.append({"suggestion": f"建议维护{d.name}"})
    return {"mode": "local", "results": results}
```
