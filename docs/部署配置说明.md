# 部署配置说明

## 端口配置

### 默认端口
- **后端服务端口**：8000
- **前端开发服务器端口**：5173

### 配置优先级（从高到低）

```
1. 系统环境变量 PORT
       │
2. .env 文件中的 SERVER_PORT
       │
3. config.py 中的默认值 (8000)
```

### 配置方式

#### 方式一：环境变量（最高优先级）

```powershell
# Windows
$env:PORT=9000
python start.py

# Linux/macOS
PORT=9000 python start.py
```

#### 方式二：.env 文件

编辑 `backend/.env`：

```ini
SERVER_PORT=8000
SERVER_HOST=0.0.0.0
```

#### 方式三：默认值

如果以上都未配置，系统默认使用端口 **8000**。

### 验证端口配置

启动服务后，控制台会显示当前使用的端口：

```
==================================================
  能耗智能管理优化平台 v1.0.0
==================================================
  服务地址: http://localhost:8000
  API 文档: http://localhost:8000/docs
  ReDoc:   http://localhost:8000/redoc
==================================================
```

## 环境变量参考

### 后端环境变量（backend/.env）

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `PORT` | - | 系统环境变量，覆盖 SERVER_PORT |
| `SERVER_PORT` | 8000 | 后端服务端口 |
| `SERVER_HOST` | 0.0.0.0 | 绑定地址 |
| `DATABASE_URL` | sqlite:///./energy_optimizer.db | 数据库连接字符串 |
| `DEBUG` | false | 调试模式开关 |
| `COZE_API_KEY` | - | Coze 智能体 API Key |
| `COZE_API_BASE` | https://api.coze.cn | Coze API 基础地址 |

### 前端环境变量（frontend/.env）

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `BACKEND_PORT` | 8000 | 后端服务端口（代理目标） |
| `BACKEND_HOST` | localhost | 后端服务地址（代理目标） |

## 常见问题

### Q: 端口被占用怎么办？

**方法一：使用环境变量更换端口**

```powershell
$env:PORT=9000
python start.py
```

**方法二：查找并停止占用进程**

```powershell
# Windows
netstat -ano | findstr :8000
taskkill /F /PID <进程ID>

# Linux/macOS
lsof -i :8000
kill -9 <进程ID>
```

### Q: 前端无法连接到后端？

1. 确认后端服务正在运行：
   ```powershell
   python -c "import httpx; print(httpx.get('http://localhost:8000/api/dashboard/overview').status_code)"
   ```

2. 确认前端代理配置正确：
   - 检查 `frontend/.env` 中的 `BACKEND_PORT` 是否为后端实际端口
   - 检查 `frontend/vite.config.js` 中的代理配置

### Q: 如何在生产环境中配置端口？

在生产环境中，建议通过环境变量或配置管理工具（如 Kubernetes ConfigMap、Docker Compose）设置端口，避免硬编码。

#### Docker Compose 示例

```yaml
services:
  backend:
    environment:
      - SERVER_PORT=8000
      - DEBUG=false
    ports:
      - "8000:8000"
```

#### Nginx 反向代理

```nginx
location /api/ {
    proxy_pass http://localhost:8000/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}
```

## 启动方式

### 开发环境

```powershell
# 后端
cd backend
python start.py

# 前端
cd frontend
npm run dev
```

### 生产环境

```powershell
# 后端（使用 uvicorn）
cd backend
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4

# 前端（构建后）
cd frontend
npm run build
# 部署 dist 目录到静态服务器
```

## 端口一致性检查

项目提供了端口一致性检查脚本：

```powershell
python scripts/check_port_consistency.py
```

该脚本会检查所有配置文件和文档中的端口引用，确保一致性。检查通过输出：

```
端口一致性检查 (期望端口: 8000)
==================================================

[OK] 所有检查文件中的端口引用一致 (端口: 8000)
```

如果发现不一致，会列出具体位置以便修正。
